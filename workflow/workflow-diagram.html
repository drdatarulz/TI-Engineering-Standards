Orchestrated Development
orchestrate-v3 — PR-Based Pipeline
▸ orchestrate-v3 — Per Ticket Pipeline
✅ Pre-Flight

Per-Ticket Setup

Read ticket, verify readiness (acceptance criteria exist), pull latest main, tag for rollback, create branch, move to In Progress.

🔄 Mode Gate

Supervised / Autonomous

Supervised: stop between tickets. Autonomous: continue, stopping only at milestones and circuit breakers.

📈 Observability

Token & Duration Tracking

Capture per-stage metrics from each sub-agent. Track review iterations, test counts, PRD amendments.

📖 Refine

refine-story-v3

Enrich issue with technical detail, codebase context, and expanded acceptance criteria. Autonomous decisions in orchestrator mode.

⚡ Implement

implement-ticket-v3 → PR #1

Fresh context. Load all standards. Explore → Plan → Implement → Unit Test → Commit → Push → Create PR. Build order: Domain → Fakes → Infrastructure → Api.

🔎 Engineering Review

engineering-review-v3 (implementation mode)

Independent context reviews PR against all 13 standards files. Posts line-level comments with standards references. Approves or requests changes.

↻ Loop: reviewer ↔ implementer fix mode — max 3 iterations
🛡️ Security Review

security-review-v3 (OWASP Top 10)

Framework-aware analysis with attack vector requirements. Every finding must explain how to exploit it — no false positives. On pass → merge PR #1.

↻ Loop: reviewer ↔ implementer fix mode — max 2 iterations
🧪 Integration Tests

integration-test-v3 → PR #2

Independent context writes tests against acceptance criteria (not developer's description). Testcontainers, real database, data isolation. Branches from updated main.

🔎 Engineering Review

engineering-review-v3 (integration-tests mode)

Reviews test quality, coverage, patterns. On approve → check off acceptance criteria → merge PR #2 → close issue.

↻ Loop: reviewer ↔ test-writer fix mode — max 3 iterations
✔️ Close

Finalize

Check off acceptance criteria, close issue. Board automation moves to Done. Delete rollback tag. Capture ticket metrics.

⬥ Milestone Gate

When a milestone-labeled issue is encountered, the orchestrator stops regardless of mode. Runs the smoke test checklist, reports results, and waits for human approval. The developer launches the app, clicks through, and validates before work continues.

🛑 Circuit Breakers

Orchestrator Halts Entirely

3 consecutive Partial/Blocked tickets. PR merge conflict. Build fails on main (rollback tag available). Review loop exhausted on 2 consecutive tickets.
