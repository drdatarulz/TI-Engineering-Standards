‚ö° Recommended Enhancements

Decisions Log
Produce alongside the PRD at end of Claude Web session. Captures "we considered X but chose Y because Z." Gives Claude Code the reasoning context, not just the requirements. Lightweight and high-value.
Screen Inventory
Conversational screen-by-screen walkthrough during PRD phase. Defines every view, its purpose, user actions, navigation paths, data dependencies, and key states. Prevents Claude Code from inferring screen boundaries ‚Äî same principle as defining compilable components.
Backlog Review Gate
10‚Äì15 min human review after Claude Code generates stories. Confirm story titles and scope map back to PRD. Catches misinterpretations before they compound through enrichment and development.
Developer Exit Checklist
Explicit verification step before the report ‚Äî not instructions at the top, but a checklist at the action boundary. Include ticket update, acceptance criteria review, and banned-dependency check. Targets the procedural drift you've observed.
Ticket Readiness Gate
Orchestrator pre-flight checks that the ticket has acceptance criteria and a non-empty description before spawning a sub-agent. Skip and log tickets that aren't ready.
Status Parse Hardening
Orchestrator explicitly parses the sub-agent STATUS line. If not exactly "Complete", "Partial", or "Blocked", treat as Partial and flag. Prevents silent misrouting if the report format drifts.
Standalone Standards Sync
Add conditional standards pull to the developer skill's standalone path (when not invoked by orchestrator). Currently only the orchestrator syncs standards.
